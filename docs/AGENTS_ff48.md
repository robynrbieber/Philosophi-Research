# Philosophi project instructions

## Purpose

This project plans a fork of `PixeroJan/obsidian-storyline` into **Philosophi**, an academic writing and research plugin aligned to Robyn's writing system.

## Working rules

- Plan before building.
- Keep StoryLine internals stable until a phase explicitly changes storage or parser behavior.
- Prefer terminology, defaults, and UI skin changes before deeper data-model changes.
- Treat the anchor as a first-class object, not a renamed scene.
- Keep docs short enough to use during implementation.
